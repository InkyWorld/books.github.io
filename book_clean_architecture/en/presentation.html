Presentation Layer

The entry point to the application. Here we accept HTTP requests, call Use Cases, and return beautiful responses.

Layer Anatomy

âœ… MUST HAVE

  • Route
    Routers/Endpoints: Defining URL paths and HTTP methods (GET, POST, PUT, DELETE).
  • Schema
    Request/Response Schemas: Pydantic models for validating incoming data and formatting responses.
  • Error
    Exception Handlers: Converting DomainError into HTTP responses (400, 404, 409).
  • Auth
    Middleware: Authentication, request logging, CORS.

â›” FORBIDDEN

  • Logic
    Business Logic: No `if order.total > 1000: apply_discount()`. This is the job of a Use Case or Entity!
  • DB
    SQL / ORM: No `session.query()` in endpoints. Everything through Use Case â†’ Repository.
  • Domain
    Returning Entities: Do not return `Order` directly. First convert to DTO/Schema.

1. Request/Response Schemas (Pydantic)

Pydantic is a data validation library. In FastAPI, it is built-in. We use it to create API "contracts".

Important: Pydantic schemas are not the same as DTOs from the Application Layer. Schemas know about JSON, serialization, HTTP. DTOs are pure data.

# presentation/schemas/order.py
from pydantic import BaseModel, EmailStr, Field, field_validator
from typing import List
from datetime import datetime

# â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•
# REQUEST SCHEMAS (Incoming Data)
# â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

class OrderItemRequest(BaseModel):
    """One item in the order"""
    name: str = Field(..., min_length=1, max_length=200, description="Item name")
    price: float = Field(..., gt=0, description="Item price (greater than 0)")
    quantity: int = Field(default=1, ge=1, le=100, description="Quantity")

    @field_validator('name')
    def name_must_not_be_empty(cls, v):
        if not v.strip():
            raise ValueError('Name cannot be empty')
        return v.strip()


class CreateOrderRequest(BaseModel):
    """Request to create an order"""
    customer_email: EmailStr  # Automatic email validation
    items: List[OrderItemRequest] = Field(..., min_length=1)
    
    class Config:
        # Example data for Swagger documentation
        json_schema_extra = {
            "example": {
                "customer_email": "user@example.com",
                "items": [{"name": "Python Book", "price": 29.99, "quantity": 1}]
            }
        }


# â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•
# RESPONSE SCHEMAS (Outgoing Data)
# â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

class OrderItemResponse(BaseModel):
    name: str
    price: float
    quantity: int
    subtotal: float  # Calculated field


class OrderResponse(BaseModel):
    """Response with order data"""
    id: str
    customer_email: str
    items: List[OrderItemResponse]
    total_amount: float
    status: str
    created_at: datetime
    
    class Config:
        from_attributes = True  # Allows creating from ORM models


class ErrorResponse(BaseModel):
    """Standard error format"""
    error: str
    detail: str
    code: str # E.g.: "ORDER_ALREADY_PAID"

2. Routers and Endpoints (FastAPI)

A router is a "thin" layer. Its task: accept data, call a Use Case, return the result. No logic!

# presentation/routers/orders.py
from fastapi import APIRouter, Depends, HTTPException, status
from typing import List
import uuid

# Import schemas (Presentation Layer)
from ..schemas.order import CreateOrderRequest, OrderResponse, ErrorResponse

# Import Use Cases and DTOs (Application Layer)
from application.use_cases import CreateOrderUseCase, GetOrderUseCase
from application.dto import CreateOrderCommand

# Import DI (dependency container)
from ..dependencies import get_create_order_use_case, get_order_use_case

router = APIRouter(prefix="/orders", tags=["Orders"])


@router.post(
    "/",
    response_model=OrderResponse,
    status_code=status.HTTP_201_CREATED,
    summary="Create new order",
    responses={
        400: {"model": ErrorResponse, "description": "Invalid data"},
        409: {"model": ErrorResponse, "description": "State conflict"}
    }
)
async def create_order(
    request: CreateOrderRequest,
    use_case: CreateOrderUseCase = Depends(get_create_order_use_case)
) -> OrderResponse:
    """
    Creates a new order.
    
    - **customer_email**: Customer's email
    - **items**: List of items (minimum 1)
    """
    
    # 1. Convert Pydantic Schema â†’ Application DTO
    command = CreateOrderCommand(
        customer_email=request.customer_email,
        items=[
            {"name": item.name, "price": item.price, "quantity": item.quantity}
            for item in request.items
        ]
    )
    
    # 2. Call Use Case (all logic IS THERE)
    result = use_case.execute(command)
    
    # 3. Convert result to Response Schema
    return OrderResponse(
        id=result.order_id,
        customer_email=result.customer_email,
        items=[
            OrderItemResponse(
                name=i['name'],
                price=i['price'],
                quantity=i['quantity'],
                subtotal=i['price'] * i['quantity']
            ) for i in result.items
        ],
        total_amount=result.total_amount,
        status=result.status,
        created_at=result.created_at
    )


@router.get("/{order_id}", response_model=OrderResponse)
async def get_order(
    order_id: uuid.UUID,
    use_case: GetOrderUseCase = Depends(get_order_use_case)
) -> OrderResponse:
    """Get order by ID"""
    result = use_case.execute(order_id)
    
    if result is None:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Order {order_id} not found"
        )
    
    return OrderResponse(**result.dict())


@router.post("/{order_id}/pay", response_model=OrderResponse)
async def pay_order(
    order_id: uuid.UUID,
    use_case: PayOrderUseCase = Depends(get_pay_order_use_case)
) -> OrderResponse:
    """Pay for order"""
    result = use_case.execute(order_id)
    return OrderResponse(**result.dict())

3. Error Handling (Exception Handlers)

The Domain throws a `DomainError`. The Presentation Layer must intercept them and turn them into understandable HTTP responses.

# presentation/exception_handlers.py
from fastapi import Request
from fastapi.responses import JSONResponse
from starlette import status

# Import domain errors
from domain.exceptions import (
    DomainError,
    InvalidEmailError,
    OrderNotFoundError,
    OrderAlreadyPaidError,
    InsufficientFundsError
)


# Mapping: Domain Error â†’ HTTP Status + Code
ERROR_MAPPING = {
    InvalidEmailError: (status.HTTP_400_BAD_REQUEST, "INVALID_EMAIL"),
    OrderNotFoundError: (status.HTTP_404_NOT_FOUND, "ORDER_NOT_FOUND"),
    OrderAlreadyPaidError: (status.HTTP_409_CONFLICT, "ORDER_ALREADY_PAID"),
    InsufficientFundsError: (status.HTTP_402_PAYMENT_REQUIRED, "INSUFFICIENT_FUNDS"),
}


async def domain_error_handler(request: Request, exc: DomainError):
    """
    Global handler for all domain errors.
    Converts DomainError into a beautiful JSON response.
    """
    # Search for mapping for a specific error type
    error_type = type(exc)
    status_code, error_code = ERROR_MAPPING.get(
        error_type, 
        (status.HTTP_400_BAD_REQUEST, "DOMAIN_ERROR")
    )
    
    return JSONResponse(
        status_code=status_code,
        content={
            "error": error_type.__name__,
            "detail": str(exc),
            "code": error_code
        }
    )


# Registration in main.py:
# app.add_exception_handler(DomainError, domain_error_handler)
Advantage: The Domain remains pure. It doesn't know about HTTP codes. And we can easily change the response format without touching the business logic.

4. Dependency Injection in FastAPI

FastAPI uses the `Depends()` system for dependency injection. This is our Composition Root.

# presentation/dependencies.py
from functools import lru_cache
from fastapi import Depends
from sqlalchemy.orm import Session

# Infrastructure
from infrastructure.db.session import get_session
from infrastructure.repositories import SqlAlchemyOrderRepository
from infrastructure.adapters.email import SmtpEmailService
from infrastructure.uow import SqlAlchemyUnitOfWork

# Application
from application.use_cases import CreateOrderUseCase, GetOrderUseCase, PayOrderUseCase

# Configuration (can be moved to a separate file)
from config import settings


@lru_cache()
def get_email_service():
    """Singleton: created once for the entire application lifetime"""
    return SmtpEmailService(
        host=settings.SMTP_HOST,
        port=settings.SMTP_PORT,
        user=settings.SMTP_USER,
        password=settings.SMTP_PASSWORD
    )


def get_unit_of_work(session: Session = Depends(get_session)):
    """Created for each request (Request Scope)"""
    return SqlAlchemyUnitOfWork(session)


def get_create_order_use_case(
    uow = Depends(get_unit_of_work),
    email_service = Depends(get_email_service)
) -> CreateOrderUseCase:
    """Assemble Use Case with all dependencies"""
    return CreateOrderUseCase(
        uow=uow,
        email_service=email_service
    )


def get_order_use_case(
    uow = Depends(get_unit_of_work)
) -> GetOrderUseCase:
    return GetOrderUseCase(uow=uow)


def get_pay_order_use_case(
    uow = Depends(get_unit_of_work),
    email_service = Depends(get_email_service)
) -> PayOrderUseCase:
    return PayOrderUseCase(
        uow=uow,
        email_service=email_service
    )

5. Application Assembly (main.py)

# main.py
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

# Routers
from presentation.routers import orders, users, payments

# Error Handlers
from presentation.exception_handlers import domain_error_handler
from domain.exceptions import DomainError


def create_app() -> FastAPI:
    """Application Factory. Allows creating different configurations for tests."""
    
    app = FastAPI(
        title="Order Service API",
        description="Order Management Service (Clean Architecture)",
        version="1.0.0",
        docs_url="/docs",
        redoc_url="/redoc"
    )
    
    # CORS for frontend
    app.add_middleware(
        CORSMiddleware,
        allow_origins=["http://localhost:3000"],
        allow_methods=["*"],
        allow_headers=["*"]
    )
    
    # Register error handlers
    app.add_exception_handler(DomainError, domain_error_handler)
    
    # Connect routers
    app.include_router(orders.router, prefix="/api/v1")
    app.include_router(users.router, prefix="/api/v1")
    app.include_router(payments.router, prefix="/api/v1")
    
    return app


# Create instance
app = create_app()


# Run: uvicorn main:app --reload

6. Alternative: Django Views

If you are using Django, the principles are the same. Just the syntax is different.

Django Class-Based View (CBV) 
# orders/views.py
from django.http import JsonResponse
from django.views import View
import json

from application.use_cases import CreateOrderUseCase
from application.dto import CreateOrderCommand
from domain.exceptions import DomainError

# DI manual (or use django-injector)
from .dependencies import get_create_order_use_case


class OrderCreateView(View):
    
    def post(self, request):
        try:
            # 1. Parse JSON
            data = json.loads(request.body)
            
            # 2. Create Command
            command = CreateOrderCommand(
                customer_email=data['customer_email'],
                items=data['items']
            )
            
            # 3. Call Use Case
            use_case = get_create_order_use_case()
            result = use_case.execute(command)
            
            # 4. Return response
            return JsonResponse(result.dict(), status=201)
            
        except DomainError as e:
            return JsonResponse({
                'error': type(e).__name__,
                'detail': str(e)
            }, status=400)
        
        except KeyError as e:
            return JsonResponse({
                'error': 'ValidationError',
                'detail': f'Missing field: {e}'
            }, status=400)

3. Business Logic in Middleware

Middleware is intended for cross-cutting concerns: logging, authentication, CORS. NOT for business rules!
