Danger Zone Common Mistakes

Main Antipatterns

How NOT to write code. Mistakes that kill projects in a year.

1. Anemic Domain (Anemic Model)

The most common sin. This is when your classes `Order`, `User` are just containers for data (getters/setters), and all the logic lies in Services.

âŒ How it looks (BAD)

Entity is dumb. Use Case is smart and does micromanagement.

# domain/entities.py
class Order:
    self.items = []
    self.total = 0
    self.status = "new"

# application/services.py
def add_item_service(order, item):
    order.items.append(item)
    order.total += item.price # Logic outside!
    if order.total > 1000:    # Logic outside!
        order.status = "vip"

âœ… How to fix (Rich Model)

Move behavior into data. "Tell, don't ask" principle.

# domain/entities.py
class Order:
    def add_item(self, item):
        self.items.append(item)
        self._recalculate_total() # Entity knows how to calculate
        self._check_vip_status()  # Entity updates status itself
