Domain Layer

The heart of your application. Pure business logic, isolated from the outside world.

What lives here?

The Domain Layer contains Enterprise Business Rules. These are rules that would exist even if you didn't have a computer. This code reflects the real world of your business.

Iron Rule:

NO external imports. No SQLAlchemy, Pydantic, Requests, Django. Only Python standard library (dataclasses, typing, datetime, uuid).

1. Value Objects

Small immutable objects. They describe properties of things. Two Value Objects are equal if their data is equal.

❌ Primitive Obsession

# Email passed as a simple string
# Validation scattered across code
def register(email: str):
    if "@" not in email:
        raise ValueError("Invalid")
    ...

✅ Value Object

@dataclass(frozen=True)
class Email:
    value: str

    def __post_init__(self):
        if "@" not in self.value:
            raise ValueError("Invalid email")

def register(email: Email):
    ...

2. Entities

Objects with unique IDs. They live through time, changing their state. Avoid Anemic Domain Model — when classes contain only data and logic lives in services.

Rich Model Example:

import uuid
from dataclasses import dataclass, field
from datetime import datetime
from typing import List

# Value Object
@dataclass(frozen=True)
class Money:
    amount: float
    currency: str = "USD"

@dataclass
class OrderLine:
    product_id: str
    quantity: int
    price: Money

# Entity
@dataclass
class Order:
    id: str = field(default_factory=lambda: str(uuid.uuid4()))
    lines: List[OrderLine] = field(default_factory=list)
    status: str = "NEW"
    
    # ⭐️ BUSINESS LOGIC LIVES HERE, NOT IN A SERVICE
    def add_line(self, product_id: str, quantity: int, price: Money):
        if self.status != "NEW":
            raise DomainException("Cannot modify confirmed order")
        
        self.lines.append(OrderLine(product_id, quantity, price))

    def mark_paid(self):
        if not self.lines:
            raise DomainException("Cannot pay for empty order")
        self.status = "PAID"

4. Domain Events

These are side-effects of business logic. An entity shouldn't send emails or write logs itself. It should say: "This happened". Events are simple DTOs. 

@dataclass(frozen=True)
class OrderPaid: # Name in past tense!
    order_id: str
    paid_at: datetime

@dataclass
class Order:
    # ... fields ...
    events: List[object] = field(default_factory=list, init=False)

    def pay(self):
        self.status = "PAID"
        # Invariant checked, state changed. Record the fact.
        self.events.append(OrderPaid(self.id, datetime.now()))

Processing these events is handled by Infrastructure (UnitOfWork) or Application layer, not the Entity itself.
