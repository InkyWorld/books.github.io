Value Objects

Small, immutable, but important domain building blocks.

Unlike an Entity, a Value Object is defined not by an identity, but by the value of its attributes.

Examples: Money, Email, Address, Coordinate, DateRange.

Characteristics:

  • Immutability: Once created, fields cannot be changed. If a change is needed, create a new object.
  • Structural Equality: Two Value Objects are equal if their values are equal (5$ == 5$).
  • Self-Validation: An object cannot exist in an invalid state.
@dataclass(frozen=True)
class Money:
    amount: Decimal
    currency: str

    def __post_init__(self):
        if self.amount < 0:
            raise ValueError("Money cannot be negative")

    def add(self, other: 'Money') -> 'Money':
        if other.currency != self.currency:
            raise ValueError("Currencies mismatch")
        return Money(self.amount + other.amount, self.currency)
