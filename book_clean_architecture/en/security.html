Auth & Permissions

How to manage users and permissions without pain.

Many tutorials show how to hash passwords, but stay silent about the main architectural question: How to pass WHO performs the action into the Use Case?

1. The Problem: Current User in Layers

If you pass request.user into a Use Case, you violate architecture (coupling to a framework). If you pass user_id into every method, the code turns into spaghetti.

Solution: Identity Provider Abstraction

1. Interface (Application Layer): Create an interface to retrieve the current user or authentication context.

# application/interfaces/identity.py
from typing import Protocol
from domain.entities.user import UserId

class IIdentityProvider(Protocol):
    def get_current_user_id(self) -> UserId:
        ...
    
    def get_permissions(self) -> list[str]:
        ...

2. Implementation (Infrastructure Layer): Implement via ContextVars (for Async) or ThreadLocal.

# infrastructure/identity/fastapi_provider.py
from contextvars import ContextVar
from application.interfaces/identity import IIdentityProvider

# Global context variable (per request)
_current_user_id: ContextVar[str] = ContextVar("current_user_id", default=None)

class ContextIdentityProvider(IIdentityProvider):
    def get_current_user_id(self) -> str:
        uid = _current_user_id.get()
        if not uid:
            raise UnauthorizedError()
        return uid

# FastAPI Middleware sets the value
async def auth_middleware(request: Request, call_next):
    token = request.headers.get("Authorization")
    user_id = decode_token(token)
    token = _current_user_id.set(user_id) # Set context
    try:
        response = await call_next(request)
    finally:
        _current_user_id.reset(token) # Clean up

2. Where to Check Permissions?

Option A: In Use Case (Business Rules)

If "Only admin can cancel an order" â€” this is a business rule.

class CancelOrderUseCase:
    def __init__(self, identity: IIdentityProvider, repo: OrderRepo):
        self.identity = identity
        
    def execute(self, order_id: str):
        user_id = self.identity.get_current_user_id()
        order = self.repo.get(order_id)
        
        if order.owner_id != user_id and "ADMIN" not in self.identity.get_permissions():
            raise PermissionDenied("Not yours!")
            
        order.cancel()

Option B: In Controller (Route Protection)

If "This URL is available only to authorized users", that is the responsibility of the Presentation layer (FastAPI Depends).
