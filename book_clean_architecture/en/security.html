Step 0 Guard at the entrance (Middleware)

This is the Infrastructure layer. Middleware intercepts the request BEFORE it reaches the business logic.

# infrastructure/web/middleware.py

# Framework calls this for EVERY request
def auth_middleware(request, next_handler):
    # 1. Stop! Who goes there? (Check the token)
    token = request.headers.get("Authorization")
    
    if not is_valid(token):
        return ErrorResponse("Access denied", 401)

    # 2. If all is good â€” remember the user ID
    # (We put it in "context" so UseCase can find it)
    user_context.set(extract_user_id(token))
    
    # 3. Go ahead! (Call the next layer)
    return next_handler(request)

Step 2 Use in Use Case

Business logic (Use Case) uses this service to check permissions.

# application/use_cases.py

class DeleteOrderUseCase:
    def __init__(self, security: SecurityService, repo: OrderRepo):
        self.security = security
        self.repo = repo

    def execute(self, order_id):
        # 1. Ask: who came?
        user_id = self.security.get_user_id()
        
        # 2. Get the order
        order = self.repo.get(order_id)

        # 3. Check: is this your order?
        if order.owner_id != user_id:
             raise Error("You cannot delete other people's orders!")
             
        self.repo.delete(order_id)
What's the magic?

Notice: our Use Case DOES NOT know about HTTP, Headers, Cookies, or Tokens. It simply asks for the user ID. This makes the code clean and simple to test.
