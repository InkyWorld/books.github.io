Strategic Distillation

Separating the wheat from the chaff. How to focus on what matters.

Your resources are limited. You cannot do everything perfectly.

Distillation is the process of continuously extracting the Core Domain from the general mass of code to focus attention on it.

How to understand if garbage got into the Core?

  • Is it difficult for you to read the core code due to the abundance of utilities?
  • Do you see code that could be found in any other project (sending emails, logging)?
  • Do business experts get bored when you discuss this?

Cleaning Methods (Refactoring)

  1. Isolate Generic Subdomains: If you're writing your own template engine â€” stop. Use Jinja2. Move it outside the Core.
  2. Create Supporting Subdomains: Move auxiliary logic (Excel import) into a separate module. Let the core simply call ImportService.run().
  3. Core Domain Manifesto: Write a document (1-2 pages) describing the essence. What exactly do we do that is unique?
