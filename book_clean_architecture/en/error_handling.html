Error Handling

How to beautifully convert Exception to HTTP 400.

Exceptions are part of your layer's interface. It's important to have a clear hierarchy.

1. Exception Hierarchy

class AppException(Exception):
    """Base class for all application errors"""
    pass

class DomainException(AppException):
    """Business logic errors (insufficient funds, etc.)"""
    pass

class ResourceNotFound(AppException):
    """Not found in repository"""
    pass

2. Centralized Error Handling (Exception Handlers)

The domain throws `DomainError`. The Presentation Layer should catch them and convert them into understandable HTTP responses. Use Case should not return `JSONResponse`!

# presentation/exception_handlers.py
from fastapi import Request
from fastapi.responses import JSONResponse
from starlette import status

# Import domain errors
from domain.exceptions import (
    DomainError,
    InvalidEmailError,
    OrderNotFoundError,
    OrderAlreadyPaidError,
    InsufficientFundsError
)


# Mapping: Domain Error â†’ HTTP status + code
ERROR_MAPPING = {
    InvalidEmailError: (status.HTTP_400_BAD_REQUEST, "INVALID_EMAIL"),
    OrderNotFoundError: (status.HTTP_404_NOT_FOUND, "ORDER_NOT_FOUND"),
    OrderAlreadyPaidError: (status.HTTP_409_CONFLICT, "ORDER_ALREADY_PAID"),
    InsufficientFundsError: (status.HTTP_402_PAYMENT_REQUIRED, "INSUFFICIENT_FUNDS"),
}


async def domain_error_handler(request: Request, exc: DomainError):
    """
    Global handler for all domain errors.
    Converts DomainError to a beautiful JSON response.
    """
    # Find mapping for specific error type
    error_type = type(exc)
    status_code, error_code = ERROR_MAPPING.get(
        error_type, 
        (status.HTTP_400_BAD_REQUEST, "DOMAIN_ERROR")
    )
    
    return JSONResponse(
        status_code=status_code,
        content={
            "error": error_type.__name__,
            "detail": str(exc),
            "code": error_code
        }
    )


# Registration in main.py:
# app.add_exception_handler(DomainError, domain_error_handler)
Benefit: The domain remains clean. It doesn't know about HTTP codes. And we can easily change the response format without touching the business logic.
