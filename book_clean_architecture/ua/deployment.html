Deployment & Protection

Защита архитектуры тестами, Docker и Makefile.

Как гарантировать, что архитектура не "сгниет" через полгода? Как упаковать это всё для продакшена?

1. Архитектурные Тесты

Документацию не читают. Пайплайн в GitLab/GitHub читают все (когда он падает). Запретим неправильные импорты.

Используем pytest-archon.

# tests/test_architecture.py
from pytest_archon import archrule

def test_domain_isolation():
    (
        archrule("domain_rule")
        .match("my_project.domain*")
        .should_not_import("my_project.infrastructure*")
        .should_not_import("my_project.application*")
        .should_not_import("sqlalchemy") 
        .should_not_import("fastapi")
        .check("my_project")
    )

2. Docker Multi-Stage Build

Ваш образ должен весить 100MB, а не 1GB. Компилируем зависимости отдельно.

# BUILDER STAGE
FROM python:3.11-slim as builder
WORKDIR /app
RUN apt-get update && apt-get install -y build-essential
COPY requirements.txt .
RUN pip wheel --no-cache-dir --no-deps --wheel-dir /app/wheels -r requirements.txt

# RUNNER STAGE
FROM python:3.11-slim
WORKDIR /app
COPY --from=builder /app/wheels /wheels
COPY --from=builder /app/requirements.txt .
RUN pip install --no-cache /wheels/*

COPY src/ src/
RUN adduser --disabled-password appuser
USER appuser

CMD ["python", "-m", "src.main"]

3. Makefile как Единая Точка Входа

Никто не помнит команды запуска тестов или линтеров. Сделайте Makefile.

test:
    pytest tests/

lint:
    ruff check src/
    mypy src/

run:
    uvicorn src.main:app --reload

up:
    docker-compose up -d
