Domain Layer

Серце програми. Чиста бізнес-логіка, ізольована від зовнішнього світу.

Що тут живе?

Доменний шар містить Enterprise Business Rules. Це правила, які існували б навіть без комп'ютера. Цей код відображає реальний світ вашого бізнесу.

Залізне правило:

ЖОДНИХ зовнішніх імпортів. Ніякого SQLAlchemy, Pydantic, Requests, Django. Лише стандартна бібліотека Python (dataclasses, typing, datetime, uuid).

1. Value Objects (Об'єкти-значення)

Малі незмінні об'єкти. Вони описують властивості речей. Два Value Object рівні, якщо їхні дані рівні.

❌ Primitive Obsession

# Email передається як рядок
# Перевірки розмазані по коду
def register(email: str):
    if "@" not in email:
        raise ValueError("Invalid")
    ...

✅ Value Object

@dataclass(frozen=True)
class Email:
    value: str

    def __post_init__(self):
        if "@" not in self.value:
            raise ValueError("Некоректний email")

def register(email: Email):
    ...

2. Entities (Сутності)

Об'єкти з унікальним ID. Вони живуть у часі, змінюють свій стан. Уникайте Anemic Domain Model (анемічної моделі) — коли класи містять лише дані, а логіка в сервісах.

Приклад багатої моделі (Rich Model):

import uuid
from dataclasses import dataclass, field
from datetime import datetime
from typing import List

# Value Object
@dataclass(frozen=True)
class Money:
    amount: float
    currency: str = "USD"

@dataclass
class OrderLine:
    product_id: str
    quantity: int
    price: Money

# Entity
@dataclass
class Order:
    id: str = field(default_factory=lambda: str(uuid.uuid4()))
    lines: List[OrderLine] = field(default_factory=list)
    status: str = "NEW"
    
    # ⭐️ БІЗНЕС ЛОГІКА ЖИВЕ ТУТ, А НЕ В СЕРВІСІ
    def add_line(self, product_id: str, quantity: int, price: Money):
        if self.status != "NEW":
            raise DomainException("Не можна змінювати оформлене замовлення")
        
        self.lines.append(OrderLine(product_id, quantity, price))

    def mark_paid(self):
        if not self.lines:
            raise DomainException("Не можна оплатити порожнє замовлення")
        self.status = "PAID"

4. Domain Events (Події Домену)

Це сайд-ефекти бізнес-логіки. Сутність не повинна сама відправляти email або писати в лог. Вона повинна сказати: "Це сталося". Події — це прості DTO. 

@dataclass(frozen=True)
class OrderPaid: # Ім'я в минулому часі!
    order_id: str
    paid_at: datetime

@dataclass
class Order:
    # ... поля ...
    events: List[object] = field(default_factory=list, init=False)

    def pay(self):
        self.status = "PAID"
        # Інваріант перевірено, стан змінено. Записуємо факт.
        self.events.append(OrderPaid(self.id, datetime.now()))

Обробкою цих подій займеться шар Infrastructure (UnitOfWork) або Application, але не сама Сутність.
